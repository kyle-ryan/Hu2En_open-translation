Community-maintained English subtitles for Hungarian political video content, generated by the Hu2En pipeline and open for correction via pull request.

| File | Description |
|------|-------------|
| `HU_PM_election_transition_2026-05-09.en.srt` | English subtitles — primary file for corrections |

| `HU_PM_election_transition_2026-05-09.whisper.json` | Raw Whisper output with word-level timestamps — for developers re-running diarisation |

## SRT format

Each subtitle entry has three parts separated by a blank line:

```
1842
00:51:22,500 --> 00:51:25,300
[SPEAKER_13] I know and understand exactly how great a responsibility this is.
```

- **Sequence number** — do not change
- **Timestamps** — do not change
- **Speaker label** — do not change
- **Text** — the only field open for correction

---

## How to correct a subtitle

1. Fork this repository
2. Edit the `.en.srt` file — text line only
3. If you also edit `en_text` in the `.json`, run the validator to confirm consistency:
   ```bash
   python .github/scripts/validate_subtitles.py
   ```
4. Open a pull request using the translation correction template

The CI workflow runs the same validator automatically on every PR.

---

## Regenerating the SRT from an edited JSON

If you correct `en_text` values in the `.json` and want a fresh `.srt`:

```bash
# from the repo root, with the Hu2En venv active
.venv/bin/vidtrans <original_video> --skip-transcribe --skip-translate --out subtitles/
```

Then rename the output to match the `HU_*` naming convention before committing.

## Using the whisper.json (developers)

The `.whisper.json` contains raw Whisper output with word-level timestamps and confidence scores. Use it to re-run diarisation without re-running Whisper (saves ~1–2 hours on a 6-hour file):

```bash
cp subtitles/HU_PM_election_transition_2026-05-09.whisper.json \
   "<video stem>.whisper.json"

.venv/bin/vidtrans video.mkv --skip-whisper
```

## Naming convention for new files

```
<ISO-3166-1-alpha-2>_<short_description>_<YYYY-MM-DD>.<lang>.srt
```

Examples:
- `HU_PM_election_transition_2026-05-09.en.srt`
- `HU_PM_election_transition_2026-05-09.hu.srt`
- `DE_parliament_budget_debate_2026-03-15.en.srt`

Language codes follow [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes).

## Loading subtitles with the video

**VLC** — drag the `.srt` onto the playing video, or *Subtitle → Add Subtitle File…*

**mpv** — place the `.srt` in the same folder with the same base name as the video; mpv loads it automatically.

**IINA (macOS)** — *File → Open Subtitle…* or drag-and-drop onto the window.

**Embed permanently with ffmpeg:**
```bash
ffmpeg -i video.mp4 -i HU_PM_election_transition_2026-05-09.en.srt \
  -c copy -c:s srt -metadata:s:s:0 language=eng \
  video_with_subs.mp4
```
